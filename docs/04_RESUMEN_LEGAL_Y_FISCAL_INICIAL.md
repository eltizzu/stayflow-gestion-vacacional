# StayFlow - Resumen Legal Y Fiscal Inicial

Ultima actualizacion: 2026-05-14

## Aviso importante

Este documento no sustituye asesoramiento legal, fiscal ni de proteccion de datos. Es una guia inicial para tomar decisiones con orden antes de vender o lanzar StayFlow.

## Regla de mantenimiento

Este documento debe actualizarse cada vez que cambie el modelo de negocio, el tipo de datos tratados, el pais objetivo, el sistema de facturacion, las integraciones, los formularios, la relacion contractual o el rol frente a los datos.

## Papel probable de StayFlow frente a los datos

En un SaaS como StayFlow pueden existir dos papeles:

### Como responsable del tratamiento

StayFlow seria responsable de los datos que usa para sus propios fines, por ejemplo:

- Datos de clientes interesados.
- Formularios comerciales.
- Usuarios registrados para gestionar la cuenta.
- Facturacion propia.
- Soporte y comunicaciones comerciales.

### Como encargado del tratamiento

StayFlow probablemente seria encargado del tratamiento respecto a los datos que el cliente introduce para gestionar su operativa, por ejemplo:

- Datos de responsables de limpieza.
- Datos de empleados o colaboradores.
- Nombres de huespedes si se incorporan.
- Telefonos, emails o notas operativas si se guardan.
- Incidencias asociadas a viviendas o reservas.

En ese caso, el cliente seria responsable del tratamiento y StayFlow trataria esos datos siguiendo instrucciones del cliente.

## Papel probable del cliente

El cliente, como empresa gestora de viviendas turisticas, probablemente seria responsable del tratamiento de los datos que decide introducir y usar en StayFlow para gestionar su actividad.

Debe tener base legal para tratar esos datos y debe informar a las personas afectadas cuando corresponda.

## Obligaciones basicas iniciales

### Proteccion de datos

- Definir que datos personales se guardan.
- Evitar guardar datos innecesarios.
- Tener politica de privacidad.
- Tener contrato de encargo de tratamiento con clientes.
- Definir medidas de seguridad.
- Definir tiempos de conservacion.
- Permitir atender derechos de acceso, rectificacion, supresion, oposicion, limitacion y portabilidad cuando aplique.
- Registrar proveedores que traten datos: hosting, email, analitica, soporte, etc.

### Seguridad

- Accesos con usuario y contrasena seguros.
- Permisos por rol.
- Copias de seguridad.
- Registro basico de actividad en fases futuras.
- Cifrado en transito mediante HTTPS.
- Minimizar datos sensibles en notas libres.

### Fiscal

- Definir si se factura como autonomo o sociedad.
- Emitir facturas correctamente.
- Aplicar IVA cuando corresponda.
- Revisar obligaciones B2B si se vende a empresas espanolas.
- Tener en cuenta factura electronica obligatoria entre empresarios y profesionales segun calendario vigente.
- Revisar obligaciones de software de facturacion/VeriFactu si StayFlow en el futuro emite facturas o incluye modulo de facturacion para clientes.

## Que conviene evitar al principio

- Guardar documentos de identidad de huespedes si no es imprescindible para el MVP.
- Crear un modulo de parte de viajeros o comunicacion a autoridades sin asesoramiento especifico.
- Incluir pagos, fianzas o facturacion del cliente demasiado pronto.
- Usar WhatsApp como canal automatizado sin revisar condiciones, consentimiento y proveedor.
- Guardar datos sensibles en campos libres.
- Prometer cumplimiento legal completo sin validacion profesional.
- Mezclar producto operativo con PMS, channel manager y facturacion avanzada desde el inicio.

## Lista de documentos necesarios

Para web comercial:

- Aviso legal.
- Politica de privacidad.
- Politica de cookies, si se usan cookies no tecnicas.
- Terminos de uso o condiciones de contratacion.
- Texto legal de formularios.

Para clientes SaaS:

- Contrato o condiciones de servicio.
- Acuerdo de encargo de tratamiento.
- Anexo de subencargados.
- SLA o condiciones de soporte, si se ofrece.
- Politica de seguridad resumida.

Para gestion interna:

- Registro de actividades de tratamiento.
- Inventario de proveedores.
- Politica de conservacion de datos.
- Procedimiento de respuesta ante incidencias.
