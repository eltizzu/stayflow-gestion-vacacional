# StayFlow - Discovery Clientes

Ultima actualizacion: 2026-05-20

## Objetivo

Usar este documento para hablar con gestores reales de alquiler vacacional y descubrir si StayFlow resuelve un problema suficientemente importante antes de construir una app completa con backend, login y datos reales.

## Perfil ideal para validar primero

- Gestor local de 5 a 25 viviendas.
- Equipo pequeno con coordinacion, limpieza, recepcion o mantenimiento.
- Uso actual de WhatsApp, llamadas, hojas de calculo, calendarios o notas.
- Dolor visible en limpiezas, check-ins, llaves, incidencias o fotos de evidencia.

## Guion de llamada

1. Entender como trabajan hoy.
2. Mostrar la vista Presentacion durante 1 minuto.
3. Hacer una demo guiada de 3 a 5 minutos.
4. Volver a preguntas concretas.
5. Cerrar con una posible prueba piloto.

## Preguntas clave

- Cuantas viviendas gestionais ahora?
- Cuantas personas participan en la operativa diaria?
- Que usais hoy para coordinar limpiezas, entradas, llaves e incidencias?
- Donde se os pierden mas cosas?
- Que fallo operativo os cuesta mas dinero o reputacion?
- Que parte de StayFlow os parece mas util?
- Que parte sobra o no entendeis?
- Que tendria que tener para que lo probarais una semana?
- Pagariais por esto si os ahorra llamadas, errores o tiempo de coordinacion?
- Que precio mensual os pareceria razonable para vuestro tamano?

## Senales positivas

- El cliente cuenta casos concretos de errores recientes.
- Pregunta si puede usarlo con su equipo.
- Quiere anadir sus viviendas o responsables.
- Compara StayFlow con su forma actual de trabajar, no con una herramienta ideal.
- Pregunta por precio, prueba piloto o integraciones.

## Senales de alerta

- El problema no le duele o ya lo tiene resuelto.
- Solo pide funciones de PMS completo: reservas, revenue, channel manager o facturacion.
- Quiere usar datos reales antes de que existan login, permisos, privacidad y backend.
- No identifica quien lo usaria cada dia.

## Cierre recomendado

"Lo que tenemos ahora es una demo interactiva para validar el flujo. Si te encaja, el siguiente paso seria hacer una prueba controlada con datos ficticios o una cartera pequena, definir que campos reales necesitas y decidir si merece la pena construir la version con acceso, base de datos y permisos."

## No prometer todavia

- Integraciones automaticas con PMS o iCal.
- Portal real para propietarios.
- Subida real de fotos.
- WhatsApp automatico.
- Facturacion.
- Uso con datos personales reales.
